Numerical Solution of Radioactive Decay Kinetics Using Scilab

A computational physics project that solves the decay differential equation with Euler and 4th-order Runge–Kutta methods, validates them against the analytical solution, and recovers the decay constant from simulated data — the same ODE-based reasoning that underlies survival analysis, disease modeling, and machine learning in health research.

§ 01Overview

A small ODE, solved carefully

Radioactive decay is governed by a first-order linear ordinary differential equation: the rate of decay is proportional to the amount of material present. Although a closed-form solution exists, the equation is an ideal setting for studying how numerical integrators behave, because every approximation can be measured against exact truth.

This project implements two integrators in Scilab — the first-order Euler method and the fourth-order Runge–Kutta method (RK4) — solves the decay equation over a fixed horizon, and compares both numerical trajectories to the analytical solution. It then treats the simulated trajectory as data and recovers the decay constant λ and half-life t½ through log-linear regression, closing the loop from model to estimate.

§ 02Physics & Theory

The governing equation

Decay is a constant-rate process: at any instant, a fixed proportion of the remaining nuclei decay per unit time. That single statement produces the entire model.

dNdt=−λN
governing ODE
first-order, linear
N(t)=N0 e−λt
analytical solution
exact reference
t½=ln 2λ
half-life
derived from λ

Here N is the number of undecayed nuclei, λ the decay constant, and N0 the initial count. The half-life follows directly from the analytical solution and is independent of N0 — a hallmark of exponential processes that reappears throughout time-to-event modeling.

§ 03Numerical Methods

Two integrators, one benchmark

Both methods advance the solution in discrete steps of size h. They differ in how they estimate the slope across each step — and that difference governs how quickly error accumulates.

First order

Euler's method

Takes a single slope estimate at the start of each step and extrapolates linearly. Simple and transparent, but global error scales with h, so accuracy degrades visibly as the step size grows.

Ni+1 = Ni + h·f(Ni)
Fourth order

Runge–Kutta (RK4)

Combines four weighted slope evaluations across each step, cancelling lower-order error terms. Global error scales with h4, tracking the analytical curve almost exactly at the same step size.

Ni+1 = Ni + (h/6)(k₁ + 2k₂ + 2k₃ + k₄)

§ 05Implementation

Core Scilab routine

The script defines the decay model once and reuses it across both integrators, then estimates λ from the resulting trajectory.

decay_kinetics.sce 
// Radioactive decay:  dN/dt = -lambda * N
lambda = 0.35;        // decay constant (1/s)
N0     = 1000;        // initial nuclei
h      = 0.1;         // step size
t      = 0:h:10;       // time grid
n      = length(t);

// Analytical (exact) solution
N_exact = N0 .* exp(-lambda .* t);

// Euler's method
N_euler = zeros(1, n);  N_euler(1) = N0;
for i = 1:n-1
    N_euler(i+1) = N_euler(i) + h * (-lambda * N_euler(i));
end

// 4th-order Runge-Kutta (RK4)
f = @(N) -lambda * N;
N_rk4 = zeros(1, n);  N_rk4(1) = N0;
for i = 1:n-1
    k1 = f(N_rk4(i));
    k2 = f(N_rk4(i) + 0.5*h*k1);
    k3 = f(N_rk4(i) + 0.5*h*k2);
    k4 = f(N_rk4(i) + h*k3);
    N_rk4(i+1) = N_rk4(i) + (h/6)*(k1 + 2*k2 + 2*k3 + k4);
end

// Recover lambda from the trajectory (log-linear fit)
p          = polyfit(t, log(N_rk4), 1);
lambda_hat = -p(1);
t_half     = log(2) / lambda_hat;
§ 06Results

Accuracy and parameter recovery

RK4 follows the analytical decay curve to within numerical precision at a moderate step size, while Euler drifts above it as error compounds. Taking the logarithm of the trajectory linearizes the exponential, so the decay constant is read directly off the slope.

Three observations summarize the study: the analytical solution provides ground truth; RK4's fourth-order accuracy makes it effectively exact here; and the recovered λ̂ matches the value used to generate the data, validating the estimation pipeline end to end.

§ 07Why It Matters Now

The same equation, in health research

The decay equation is not an isolated curiosity. Its structure — a quantity changing at a rate proportional to itself — is the mathematical backbone of how modern quantitative health research describes risk, time, and dynamics over populations.

Survival analysistime-to-event
S(t) = e−λt
The exponential survival function is identical in form to the decay solution; the decay constant becomes a constant hazard rate.
Hazard modelinginstantaneous risk
dS/dt = −h(t)·S
Survival declines in proportion to the population still at risk — the same constant-rate logic that drives decay, generalized to time-varying hazards.
Disease modelingSIR / compartmental
dC/dt = −γC + …
Compartmental epidemic models are coupled ODE systems solved with the very integrators used here — Euler and Runge–Kutta.
Pharmacokineticsdrug elimination
dc/dt = −ke·c
First-order drug clearance is exponential decay by another name; elimination half-life mirrors radioactive half-life exactly.
Machine learninghealth outcomes
∂L/∂θ → numerical solvers
Gradient-based optimization, neural ODEs, and discrete-time hazard networks rest on the same numerical-integration and parameter-estimation foundations.

Solving a decay ODE by hand and recovering its rate from data is, in miniature, the workflow of survival modeling, disease dynamics, and pharmacoepidemiology. This early project established the ODE-based reasoning and estimation habits that now carry directly into quantitative health research.
